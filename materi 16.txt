penggunaan foreign key kedalam migrate ==================================

references {model dan key}

references = perintah / penggunaan referensi tabel yang akan di tuju

model = tabel / model yang dituju

key = sebagai init foreign key dan valuenya mengikuti berdasarkan tabel yang akan di relasikan

example tabel 1 punya id lalu di tabel 2 terdapat column bernama addres, addres di init sebagai foreign key yang 
valuenya berdasarkan id dari tabel 1.

ondelete = cascade
onupdate = cascade



4 pengaturan didalam foreign key : ==========================

1. RESCRICT 	= menolak perubahan tabel => ada relasi antara tabel A dan B namun yg di ubah hanya tabel A maka 
		  			     perubahan itu tidak diperbolehkan selama data tabel A masih mempunyai 
					     relasi ke data tabel B.

2. CASCADE 	= sama-sama berubah => 	ketika perubahan terjadi pada salah satu tabel maka tabel yang mempunyai relasi
				       	akan ikut berubah (perubahan pada baris).

3. SET NULL 	= mengkosongkan	=> ketika terjadi perubahan pada tabel A maka baris/data yang ada pada tabel B akan menjadi
				   NULL dengan syarat kolomnya tidak mempunyai setting NOT NULL.

4. NO ACTION 	= tidak berubah => tidak akan merubah apapun meski salah satu table ada yg dirubah ( standar SQL ).
